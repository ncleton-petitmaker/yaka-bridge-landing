L'IA, au prix que vous payez déjà.

La plupart des outils facturent l'intelligence artificielle à l'usage : chaque tâche ajoute une ligne sur la facture. Yaka-Bridge fait autrement, avec ChatGPT Codex ou un modèle local.

Votre IA travaille déjà pour vous. Autant qu'elle travaille pour votre entreprise.

Le Bridge est une petite application installée sur le poste. Quand un module a besoin d'intelligence, il dialogue avec l'IA choisie : ChatGPT Codex ou un modèle local lancé sur la machine.
D'habitude Chaque automatisation IA passe par des crédits facturés à l'usage. Plus vos équipes s'en servent, plus la facture grimpe, et personne n'ose vraiment l'utiliser.
Avec Bridge Le travail est confié à l'IA déjà disponible sur le poste. Pas de coût caché qui apparaît à la fin du mois, et une option 100 % locale si le besoin le demande.
Résultat Vos équipes essaient, recommencent, automatisent, sans se demander combien coûte chaque clic. L'IA devient un réflexe, pas un budget.

Et si vous voulez tout garder sur le PC ?

Yaka-Bridge peut aussi communiquer avec un modèle local lancé dans LM Studio. Les données restent sur la machine, et aucun abonnement IA n'est nécessaire pour ces traitements.

LM Studio comme moteur local

LM Studio lance un serveur sur le poste, puis Bridge lui parle comme à une API compatible OpenAI. Pour l'utilisateur, le module continue de fonctionner pareil : seul le moteur change.

Le bon choix dépend surtout de la VRAM : laptop, station de travail ou serveur local n'ont pas le même terrain de jeu. Les familles les plus sérieuses à surveiller sont :

Qwen3.5 9B / Qwen3.6 35B-A3B pour le meilleur compromis local : multilingue, raisonnement, vision, outils et long contexte. Qwen3-Coder-Next pour les agents de code locaux, le travail sur dépôt et les workflows de développement. Gemma 4 E2B / E4B pour les tâches rapides sur laptop, avec 26B A4B / 31B à réserver aux machines plus solides. Ministral 3 8B / 14B pour un modèle compact, multimodal, francophone et vraiment déployable en local. Phi-4 Reasoning Vision 15B pour du multimodal compact quand il faut raisonner sans passer sur un très gros modèle. Mistral Small 4 pour un serveur local plus musclé : instruction, raisonnement, code et agents dans un modèle unifié. Llama 4 Scout, Kimi K2.6, DeepSeek V4 à surveiller côté workstation ou serveur local, pas comme choix par défaut sur un PC standard.

Quand c'est suffisant

Classement de documents, extraction de champs, résumé de mails, brouillons de réponse, reformulation, contrôle de cohérence, aide sur des tickets, achats ou stocks : un bon modèle local suffit souvent. Pour les raisonnements longs, le code complexe ou les décisions sensibles, ChatGPT Codex reste le meilleur cran au-dessus.

Pourquoi ChatGPT Codex ?

Pour les tâches difficiles, l'abonnement OpenAI garde un avantage simple : il peut travailler avec vos outils au lieu de rester enfermé dans une interface séparée.

Qu'est-ce que Yaka-Bridge ?

Un ERP sur mesure qui réunit vos modules métier dans une application simple et les fait communiquer avec une IA sur votre poste.

Est-ce compatible avec ChatGPT Codex ?

Oui. Les modules peuvent utiliser ChatGPT Codex quand une tâche demande une capacité IA avancée.

Peut-on utiliser un modèle local ?

Oui. Bridge peut communiquer avec un modèle local lancé dans LM Studio, par exemple Qwen3.5, Qwen3.6, Gemma 4, Ministral 3, Phi-4 Reasoning Vision, Llama 4 Scout ou un modèle serveur comme Kimi K2.6 et DeepSeek V4.

Quand le local suffit-il ?

Classement, extraction, résumé, brouillons, reformulation et contrôles de cohérence sont souvent de bons cas d'usage locaux.
